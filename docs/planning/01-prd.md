# Working Hub Manager PRD

## MVP Capsule
1. 목표: 종합자산관리 회사 직원의 엑셀 기반 반복 업무를 웹 자동화하고, 브랜드 일관성 있는 콘텐츠를 제작하는 중앙 허브 플랫폼
2. 페르소나: 종합자산관리 회사 직원 (영업/마케팅/관리 부서, 현재 엑셀 작업 90%)
3. 핵심 기능: FEAT-1 Working Hub 대시보드, FEAT-2 업무자동화 프로그램 (수당정산 2종 + IRP 관리), FEAT-3 주식/ETF 추천, FEAT-4 콘텐츠 제작 (카드뉴스/보고서/표지)
4. 성공 지표 (노스스타): 엑셀 수작업 시간 50% 이상 절감
5. 입력 지표: 프로그램 일일 사용 횟수, 보고서 생성 건수
6. 비기능 요구: 금융 데이터 계산 정확성 (수당/수익률), 개인정보 보안
7. Out-of-scope: 재무계산기, 자산관리(B그룹), 세무/절세(C그룹)
8. Top 리스크: 증권사 웹 크롤링 보안 및 접근성 제한
9. 완화/실험: 크롤링 불가 시 엑셀 업로드 대체 경로 제공
10. 다음 단계: /screen-spec 실행

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | Working Hub Manager |
| 핵심 가치 | 분산된 엑셀 업무를 하나의 웹 허브에서 자동화, 브랜드 일관성 콘텐츠 생산 |
| 타겟 사용자 | 종합자산관리 회사 직원 (병의원 원장 대상 자산관리) |
| 간접 수혜자 | 병의원 원장 고객 (콘텐츠/보고서 수신자) |
| 비즈니스 모델 | 내부 사용 도구 (자사 직원 전용) |

### 현재 고통 (As-Is)
- 엑셀, 한글, PPT 등 프로그램을 따로따로 열어서 작업
- 엑셀 작업이 전체 업무의 90% 차지
- 수수료 계산, 포트폴리오 수익률 관리, 법인 지출관리 모두 수작업
- 콘텐츠 제작 시 회사 브랜드 일관성 유지 어려움

### 목표 상태 (To-Be)
- 하나의 웹 허브에서 모든 프로그램 접근
- 데이터 업로드 → 자동 계산 → AI 분석 → 보고서 출력 자동화
- 회사 브랜드(컬러/로고/서체) 자동 적용된 콘텐츠 제작

## 2. 핵심 기능

### FEAT-1: Working Hub 대시보드 (우선순위: 1)
- **설명**: 모든 프로그램에 접근하는 중앙 런처/허브
- **Why**: 직원들이 여러 프로그램을 따로 열어야 하는 비효율 해소
- **MVP**: 카테고리별 아코디언 + 카드 그리드 레이아웃
- **카테고리**: 업무 자동화 / 투자 분석 / 콘텐츠 제작
- **확장성**: 향후 프로그램 추가를 위한 구조

### FEAT-2: 업무 자동화 프로그램 (우선순위: 2)

#### FEAT-2-1: Dr.GM 수당정산 계산기
- **설명**: 엑셀 업로드 → 수당 계산 → 개인별 수당명세서 다운로드
- **입력**: 엑셀 파일 업로드
- **처리**: 수당 자동 계산
- **출력**: 개인별 수당명세서 다운로드

#### FEAT-2-2: 증권사 수당정산 계산기
- **설명**: 웹 크롤링 or 엑셀 업로드 → 템플릿 적용/계산 → 보고서 다운로드
- **입력**: 증권사 웹 크롤링 (보안 이슈 시 엑셀 업로드 fallback)
- **처리**: 템플릿 적용 + 수당 계산
- **출력**: 명세서/보고서 다운로드

#### FEAT-2-3: 연금저축/IRP 포트폴리오 수익률 관리기
- **설명**: 크롤링/업로드 → 수익률 분석 + 추천상품 + 리밸런싱 → 보고서 PDF
- **입력**: 웹 크롤링 or 엑셀 업로드
- **처리**: 템플릿 적용 + 직접 값 입력/수정 + AI 분석 (리밸런싱, 추천상품)
- **출력**: 보고서 미리보기 + PDF 다운로드

### FEAT-3: 주식/ETF 추천 프로그램 (우선순위: 2)
- **설명**: 증권사 API 테마 분석 → 유망 테마 선택 → Top 5 주식 추천 → 포트폴리오 담기
- **Step 1**: 증권사 API로 테마 정보 불러오기 → AI 뉴스분석으로 점수 책정 → 추천 테마 표시 → 복수 테마 바구니 담기
- **Step 2**: 선택 테마의 주식 목록 (Top 5 구별 표시) + 기간별 수익률/기관/외국인 매수 정보 → 포트폴리오 or 회사풀 담기
- **분석 보고서**: 개별 주식 분석 보고서 (팝업, NotebookLM식 리서치 기반 작성)
- **연동**: IRP 포트폴리오 관리기에 반영 가능

### FEAT-4: 콘텐츠 제작 (우선순위: 3)
- **공통 패턴**: 주제/내용 입력 → AI 글 작성 → 검토/승인 → 생성(브랜드 자동 적용) → 수정/재생성 → 다운로드
- **핵심 가치**: 회사 컬러, 로고, 디자인, 서체가 자동 적용된 일관된 브랜드 콘텐츠
- **수정 방식**: 텍스트 변경 재생성, 이미지 부분 수정 요청 재생성

#### FEAT-4-1: 카드뉴스 제작 (1순위)
- 고객에게 정보 제공 목적

#### FEAT-4-2: 보고서 제작 (2순위)

#### FEAT-4-3: 표지/홍보페이지 디자인 (3순위)

## 7. 가설 vs 사실

| 항목 | 가설/사실 | 검증 방법 |
|------|----------|----------|
| 엑셀 작업이 업무의 90% | 사실 (사용자 증언) | - |
| 웹 자동화가 시간을 절감할 것 | 가설 | 파일럿 테스트로 시간 측정 |
| 증권사 크롤링이 기술적으로 가능 | 가설 | 기술 검증 (PoC) |
| AI 콘텐츠가 품질 기준을 충족할 것 | 가설 | 샘플 생성 후 직원 피드백 |
| 브랜드 일관성이 자동 적용 가능 | 가설 | 디자인 시스템 프로토타입 |
