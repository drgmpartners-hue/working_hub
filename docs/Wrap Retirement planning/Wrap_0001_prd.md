# Wrap Retirement (랩 은퇴설계) PRD

## MVP Capsule
1. 목표: 각종 금융상품을 활용한 은퇴자금을 계획하고, 실제 투자 성과를 매년 추적하며, 계획 대비 괴리를 분석하여 고객의 성공적인 은퇴 달성을 관리하는 도구
2. 페르소나: 종합자산관리 회사 직원 (병의원 원장 고객의 은퇴설계 관리)
3. 핵심 기능: FEAT-1 희망은퇴플랜(목표 설정), FEAT-2 투자흐름(랩 투자기록), FEAT-3 은퇴플랜(상세 시뮬레이션), FEAT-4 인터랙티브계산기(계획vs실제 비교+AI가이드), FEAT-5 연금수령계획(모으기+쓰기 통합)
4. 성공 지표 (노스스타): 고객별 은퇴플랜 달성률 (계획 대비 실제 괴리율 감소)
5. 입력 지표: 월간 투자기록 입력 건수, 고객별 연간 플랜 리뷰 횟수
6. 비기능 요구: 복리 계산 정확성, 금융 데이터 보안
7. Out-of-scope: 실시간 주가 연동, 증권사 API 자동 크롤링, 세금 계산
8. Top 리스크: 복리 계산 로직 오류로 인한 잘못된 은퇴플랜 안내
9. 완화/실험: 엑셀 수식 대조 검증 + 단위 테스트 커버리지 확보
10. 다음 단계: /screen-spec 실행

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | Wrap Retirement (랩 은퇴설계) |
| 핵심 가치 | 각종 금융상품을 활용한 은퇴 계획과 실제 투자 성과를 지속적으로 비교·추적하며, 괴리 발생 시 AI 기반 조정 가이드를 제공하는 동적 은퇴설계 도구 |
| 타겟 사용자 | 종합자산관리 회사 직원 (병의원 원장 대상 자산관리) |
| 간접 수혜자 | 병의원 원장 고객 (은퇴설계 보고서 수신자) |
| 위치 | Working Hub > 카테고리 2 (투자 분석) |

### 현재 고통 (As-Is)
- 엑셀로 은퇴플랜을 관리하여 매년 수작업으로 실제 투자 결과를 업데이트
- 계획 대비 괴리가 발생해도 정량적 분석 없이 감에 의존
- 금융상품 투자의 상품 간 연결고리(exit→재투자)를 체계적으로 추적하기 어려움
- 고객별 은퇴플랜을 일관된 형식으로 관리하지 못함

### 목표 상태 (To-Be)
- 5개 탭 구조로 은퇴설계 전체 라이프사이클 관리
- 실제 투자 성과가 계획에 자동 반영되어 괴리율 실시간 확인
- AI가 괴리 해소를 위한 구체적 수치 가이드 제공 (복리계산 기반)
- 고객별 데이터 독립 관리 + 연간 보고서 생성

## 2. 핵심 기능

### FEAT-1: 희망 은퇴플랜 (1번탭, 우선순위: 1)
- **설명**: 매월 수령 희망 은퇴금액을 기반으로 목표 은퇴자금을 역산하고, 필요한 일시납/적립금액을 계산하는 표
- **Why**: 고객의 은퇴 목표를 명확한 숫자로 정의해야 이후 모든 계획의 기준선이 됨
- **형태**: 이해를 돕기 위한 표 형태로 구현
- **입력**: 희망 월 수령액, 은퇴 기간
- **출력**: 목표 은퇴자금, 필요 일시납/적립금액 계산표 + 복리 성장 그래프
- **데이터 흐름**: 저장 시 3번탭(은퇴플랜), 4번탭(인터랙티브계산기)에 반영

### FEAT-2: 투자흐름 (2번탭, 우선순위: 1)
- **설명**: 각종 금융상품(랩어카운트 등) 운용 결과를 기록하고 추적
- **Why**: 실제 투자 기록이 있어야 계획 대비 성과를 비교할 수 있음
- **2개 영역**:
  - **(1) 연간 투자흐름표**: 투자기록의 연도별 요약 (연도/연차/나이/일시납/연적립/총납입/연간총수익/연간평가액/연수익률/인출액/누적인출액/총평가금액)
  - **(2) 투자기록**: 개별 투자 상품의 시작~종료, 수익률, 추가적립, 연결상품 추적
- **UI**: 테이블 메인 + 타임라인 뷰 토글 (하이브리드)
- **핵심 기능**: 상품 간 연결고리 추적 (exit → 다음 투자로 연결)
- **데이터 흐름**: 연간 투자흐름표 → 4번탭에 반영

### FEAT-3: 은퇴플랜 (3번탭, 우선순위: 1)
- **설명**: 상세 기본정보를 입력하여 연도별 예상 평가금액을 시뮬레이션
- **Why**: 1번탭의 단순 목표를 구체적인 연도별 시뮬레이션으로 발전시킴
- **기본정보 입력**: 현재 나이, 일시납입금액, 연적립금액, 납입기간, 물가상승률&상속재원 고려, 연수익률, 목표은퇴자금, 목표 연금액, 희망 은퇴나이, 가능 은퇴나이
- **출력**: 연도별 예상 평가금액 테이블 + 성장 그래프
- **데이터 흐름**: 4번탭, 5번탭의 기준 그래프로 사용

### FEAT-4: 인터랙티브 계산기 (4번탭, 우선순위: 1)
- **설명**: "롤링 은퇴 플랜(Rolling Retirement Plan)" - 3번탭 계획 + 2번탭 실제 데이터를 결합
- **Why**: 계획과 실제의 괴리를 시각화하고, AI가 구체적 조정 방안을 제시
- **동작 방식**:
  - 3번탭 표가 기본 베이스
  - 매년 12/31 기준 2번탭의 연간 투자흐름표 데이터를 반영
  - 직전 연도까지 = 실제 데이터, 이후 = 수정 예측 그래프
  - 3번탭 기준 그래프와 겹쳐서 이격 표시
- **AI 가이드**: 목표 하회 시 복리계산 기반 수치 가이드 + 시장상황/전망/고객의지를 고려한 근거 설명 (종합금융자산관리사 에이전트)
- **핵심**: 거치/적립 투자금액, 목표수익률, 투자기간 조정 제안

### FEAT-5: 연금수령 계획 (5번탭, 우선순위: 2)
- **설명**: 모으는 기간과 쓰는 기간을 통합하여 전체 은퇴 라이프사이클을 시각화
- **Why**: 은퇴자금을 모으는 것뿐 아니라 어떻게 수령하는지까지 계획해야 완전한 은퇴설계
- **3개 영역**:
  - **(1) 모으는 기간 요약**: 모으는 기간의 핵심 지표 요약
  - **(2) 연금지급방법 비교**: 종신형, 확정형, 상속형
  - **(3) 통합 그래프**: 모으는 기간 + 쓰는 기간 연속 그래프, 연금지급방법 선택에 따라 동적 변경

### 부가 기능: 랩어카운트 관리 (Hub Data관리 메뉴)
- **설명**: Hub 바탕화면 Data관리에 '랩어카운트 관리' 메뉴 추가
- **관리 정보**: 상품명, 증권사, 투자대상, 목표수익률 등 상품 기본정보
- **용도**: 2번탭 투자기록에서 상품 선택 시 드롭다운으로 활용

## 5. MVP 범위

### Phase 1 (MVP)
- 1번탭: 희망 은퇴플랜 (목표 설정 + 복리 성장 그래프)
- 2번탭: 투자흐름 (연간 투자흐름표 + 투자기록 테이블)
- 3번탭: 은퇴플랜 (기본정보 입력 + 연도별 시뮬레이션 + 그래프)
- 4번탭: 인터랙티브 계산기 (계획vs실제 비교 그래프 + AI 가이드)
- 5번탭: 연금수령 계획 (모으기+쓰기 통합 그래프 + 지급방법 비교)
- 고객 선택 + 상단 고정 정보
- 랩어카운트 관리 (Hub Data관리 메뉴)

### Phase 2 (향후 확장)
- 타임라인 뷰 (2번탭 토글)
- 연간 보고서 PDF 자동 생성
- 고객별 알림 (연간 리뷰 시점, 괴리율 임계치 초과)
- 다중 시나리오 비교 (낙관/중립/비관)

## 6. 벤치마킹

| 유사 서비스 | 차별점 |
|------------|--------|
| 증권사 은퇴설계 시뮬레이터 | 단회성 계산 vs **매년 실제 성과 반영하는 동적 추적** |
| 엑셀 은퇴플랜 시트 | 수작업 업데이트 vs **투자기록 자동 연동 + AI 가이드** |
| 로보어드바이저 | 자동 운용 vs **금융상품 수동 운용 + 기록/분석 지원** |

## 7. 가설 vs 사실

| 항목 | 가설/사실 | 검증 방법 |
|------|----------|----------|
| 매년 실제 성과를 입력할 것 | 가설 | 파일럿 고객 3명으로 1년 운영 |
| AI 가이드가 실제 투자 결정에 도움 | 가설 | 직원 피드백 수집 |
| 계획 대비 괴리 시각화가 고객 설득에 유효 | 가설 | 고객 미팅 후 전환율 측정 |
| 복리 계산 로직이 엑셀과 동일한 결과 | 사실 | 단위 테스트로 검증 가능 |
