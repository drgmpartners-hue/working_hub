# IRP/연금저축 포트폴리오 관리 — 기능 기획서

> 작성일: 2026-03-16
> 대상: 기존 Working Hub Manager의 IRP/연금저축 포트폴리오 페이지 전면 재설계

---

## 1. 개요

### 목표
- 증권사 캡처 이미지 → AI 데이터 추출 → 포트폴리오 분석 → 브랜드 보고서 PDF까지 완전 자동화
- 1~3단계를 연결해서 순차 진행하거나, 각 단계를 독립적으로 실행 가능

### 핵심 문제 (현재)
- 이미지 붙여넣기(Ctrl+V)가 작동하지 않음 (onClick → 파일 다이얼로그 열림 → div 포커스 이탈)
- 위험도, 지역 데이터 컬럼 없음
- 보고서 구성 미완성

---

## 2. 3단계 흐름

```
[1단계] 고객 선택 + 이미지 첨부 → AI 추출 → DB 저장
   ↓ (연결 or 독립)
[2단계] 이력 데이터 조회 → 분석 화면 확인
   ↓ (연결 or 독립)
[3단계] 보고서 생성 → PDF 다운로드
```

---

## 3. 1단계: 데이터 수집

### 3-1. 고객 선택/입력
- **기존 고객**: DB 검색 → 선택 (이름, 계좌번호 자동완성)
- **신규 고객**: 직접 입력 (이름, 계좌유형, 계좌번호)
- **배치 처리**: 여러 고객을 한 번에 추가하여 일괄 처리

### 3-2. 이미지 첨부 (핵심 버그 수정 포함)
| 방법 | 설명 |
|------|------|
| Ctrl+V 붙여넣기 | window 레벨 paste 이벤트 수신 → 포커스 무관하게 작동 |
| 파일 선택 | 클릭 → 파일 다이얼로그 (별도 버튼으로 분리) |

> **버그 수정**: `onPaste`를 div 레벨에서 `window` 레벨로 올려서 포커스 무관하게 Ctrl+V 작동

### 3-3. AI 데이터 추출
- 이미지 → Gemini Vision API → 아래 데이터 추출
- 추출 실패 항목은 수동 입력란으로 fallback

#### 추출 데이터 구조
**계좌 정보 (1회)**
| 필드 | 설명 |
|------|------|
| 납입원금 | 누적 납입 원금 |
| 월납입액 | 월 납입 금액 |
| 예수금 | 현금 잔고 |
| 총자산 | 전체 평가자산 |
| 매입금액 | 총 매입 금액 |
| 평가금액 | 현재 평가 금액 |
| 평가손익 | 평가금액 - 매입금액 |
| 총수익률 | 전체 수익률 (%) |

**투자상품 정보 (상품별 반복)**
| 필드 | 설명 |
|------|------|
| 상품유형 | 주식형/채권형/혼합형/MMF 등 |
| 상품명 | 펀드/ETF명 |
| 종목코드 | 선택 |
| 잔고수량 | 보유 수량 |
| 매입가 | 평균 매입가격 |
| 현재가 | 현재 가격 |
| 평가금액 | 현재 평가 금액 |
| 평가손익 | 손익 금액 |
| 수익률 | 수익률 (%) |
| 만기일 | 해당 시 |
| **위험도** | 1~5등급 또는 저/중/고 |
| **지역** | 국내/미국/글로벌/신흥국 등 |

## 4. 2단계: 분석 화면

### 4-1. 고객 이력 조회
- 저장된 고객 선택 → 날짜별 스냅샷 목록 표시
- 보고서로 만들 스냅샷(기준일) 선택

### 4-2. 포트폴리오 표
- 계좌 요약 정보 + 투자상품별 상세 테이블
- 컬럼: 상품명 / 상품유형 / **지역** / **위험도** / 매입금액 / 평가금액 / 수익률 / 만기일

### 4-3. 분석 차트 (미리보기)
| 차트 | 설명 |
|------|------|
| 기간별 수익률 | 3개월 / 6개월 / 1년 선택 탭, 라인 차트 |
| 지역 분산 | 투자 지역별 비중 파이/도넛 차트 |
| 위험도 분산 | 위험등급별 비중 바/도넛 차트 |

> **데이터 소스**: 1단계에서 날짜별로 저장된 스냅샷 이력 → 날짜 기준으로 그래프 구성

## 5. 3단계: 보고서

```

[포트폴리오 분석]

[포트폴리오 변경 안내]
 - 현재 구성 분석 기반 리밸런싱 제안

```

### 5-2. 보고서 형식
- 고정 템플릿 (회사 로고/컬러 자동 적용)

### 5-3. 배치 처리
- 여러 고객 선택 → 보고서 일괄 생성

## 8. 추가 기능: 고객 확인 포털 + 리밸런싱 제안

### 8-1. 두 가지 링크 모드

> "항상 볼 수 있는 창구" + "지금 이 순간, 당신을 위한 제안"

| 모드 | URL | 유효기간 | 내용 |
|------|-----|---------|------|
| **조회 링크** | `/client/[token]` | 영구 | 날짜 선택 → 포트폴리오 현황 확인 |
| **제안 링크** | `/client/[token]?suggest=[id]` | **7일** | 현황 + 추천 변경안 + 통화예약 CTA |

### 8-2. 조회 링크 흐름 (기본)

```
고객이 링크 클릭 → 이름/생년월일/핸드폰 인증
  ↓
계좌 탭 선택 (IRP / 연금저축 / 연금저축_거치)
  ↓
스냅샷 날짜 선택 (업로드된 날짜만)
  ↓
포트폴리오 현황 보고서:
  - 계좌 개요 (자금현황)
  - 포트폴리오 분석 표
  - 지역/위험도 분산 차트
  - 기간별 수익률 그래프
  - AI 코멘트 (최신 날짜에만)
```

### 8-3. 리밸런싱 제안 링크 흐름 (직원이 적극 발송)

```
[직원 - 2단계 분석 화면]
  포트폴리오 표에서 수정비중 인라인 편집
    예: 'AB미국그로스 펀드' 10.13% → 5%
    합계=100% 자동 검증

  "리밸런싱 제안 발송" 버튼 클릭
    ├── 링크 복사 (카카오톡/문자용)
    └── 이메일 자동 발송
    → 제안 ID 생성 (7일 후 만료)

[고객 - 제안 링크 접속]
  인증 후 화면 구성:
    ┌─────────────────────────────┐
    │  현재 포트폴리오             │
    │  (날짜별 현황 확인 가능)     │
    ├─────────────────────────────┤
    │  ✨ 추천 포트폴리오 변경안   │  ← 제안 링크에만 표시
    │  상품명 | 현재비중 | 추천비중 │
    │  AB미국그로스 10.1% → 5%   │
    │  ...                        │
    │  AI 코멘트 (변경 이유 설명) │
    ├─────────────────────────────┤
    │  📞 담당자와 상담하기        │  ← CTA
    │  [통화 예약하기]             │
    └─────────────────────────────┘

[통화 예약]
  날짜 선택 (기본값: 오늘)
  시간대 선택 (예: 오전 10시 / 오후 2시 / 오후 4시)
  → 직원에게 이메일 알림
  → 고객에게 예약 확인 문자/이메일

[3일 후 만료]
  제안 영역: "제안 기간이 만료되었습니다"
  조회 기능: 그대로 유지
```

### 8-4. 직원의 2단계 비중 수정 UX

```
포트폴리오 표 (인라인 편집)
─────────────────────────────────────────────
NO | 상품명          | 현재비중 | [수정비중] | 변경후 평가금액 | (+/-)
 1 | AB미국그로스     | 10.13%  | [5.00% ▾] |    6,000,000  | -4,812
 2 | 우리하이플러스   | 21.56%  | [25.00%▾] |   30,682,500  | +5,195
   ...
   합계             | 100%    | [100%✓]   |               |
─────────────────────────────────────────────
                              [리밸런싱 제안 발송] 버튼
```

- 수정비중 합계 ≠ 100% → 빨간 경고, 발송 버튼 비활성화
- 변경후 평가금액, (+/-) 자동 계산

### 8-5. 고객 정보 확장

| 추가 필드 | 용도 |
|----------|------|
| `birth_date` | 인증 수단 |
| `phone` | 인증 + 통화 예약 알림 |
| `email` | 이메일 자동 발송 |
| `portal_token` | 조회 링크 토큰 (영구) |

### 8-6. 보안

| 항목 | 방식 |
|------|------|
| 조회 링크 | `/client/[랜덤 UUID]` |
| 2차 인증 | 이름 + 생년월일 + 핸드폰 |
| 3회 실패 | 30분 잠금 |
| 제안 링크 | 토큰 + 제안 ID, 7일 만료 후 제안 영역 숨김 |

### 8-7. AI 코멘트 규칙

| 상황 | AI 코멘트 |
|------|----------|
| 조회 링크 - 최신 날짜 | 포트폴리오 분석 코멘트 표시 |
| 조회 링크 - 과거 날짜 | 코멘트 없음 |
| 제안 링크 | 변경 이유 + 기대 효과 코멘트 표시 |

### 8-3. 고객 정보 확장

| 추가 필드 | 용도 |
|----------|------|
| `birth_date` | 인증 수단 |
| `phone` | 인증 수단 |
| `email` | 이메일 자동 발송 |
| `portal_token` | 고객별 고유 링크 토큰 (UUID, 영구 유효) |

### 8-4. 보안

| 항목 | 방식 |
|------|------|
| URL | `/client/[랜덤토큰]` — 추측 불가 |
| 2차 인증 | 이름 + 생년월일 + 핸드폰 일치 확인 |
| 3회 실패 | 30분 잠금 |
| 데이터 범위 | 해당 고객의 계좌/스냅샷만 노출 |

### 8-5. AI 코멘트 규칙

- **최신 날짜 스냅샷 선택 시**: AI 코멘트 표시
- **과거 날짜 스냅샷 선택 시**: AI 코멘트 숨김 (과거 분석은 무의미)

### 8-6. 모바일 최적화

- 반응형 레이아웃 (모바일 우선)
- 차트는 스와이프로 전환
- 빠른 로딩 (차트 lazy load)
