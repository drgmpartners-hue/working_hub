# 시황 분석 기반 AI 코멘트 시스템

## 아키텍처

```
[AI 코멘트 생성 클릭]
    ↓
① 종목별 핵심 키워드 추출
   KODEX 미국빅테크 → "미국 빅테크"
   미래에셋인도중소형 → "인도증시"
    ↓
② 키워드별 시황 분석 (DB 캐시 48시간)
   ┌─ DB에 캐시 있음? → 재사용
   └─ 캐시 없음? → 네이버 뉴스 5건 검색
                    → Gemini: "이 뉴스로 시황 분석 작성해줘"
                    → DB 저장 (48시간)
    ↓
③ 코멘트 조합
   [분석 코멘트] 포트폴리오 데이터 + 키워드별 시황 → Gemini
   - 수익/손실 원인을 시장 상황과 연결
   - 강점, 리스크, 주의사항 포함
   - 8-12줄 분량

   [변경 코멘트] 변경 내역(매도/축소/확대/신규) + 키워드별 시황 → Gemini
   - 왜 매도했는지, 왜 비중 늘렸는지 설명
   - 변경 후 기대 효과
   - 10-15줄 분량
```

## 캐시 재활용

```
고객A (09:00) → 키워드: 미국 빅테크, 인도증시, 로봇주
  → 3개 시황 분석 신규 생성 + DB 저장

고객B (09:30) → 키워드: 미국 빅테크, 채권시장
  → "미국 빅테크" 캐시 재사용, "채권시장"만 신규 생성

고객C (10:00) → 키워드: 인도증시, 로봇주
  → 2개 모두 캐시 재사용 (API 호출 0건)
```

## 키워드 추출 규칙

종목별 1개 핵심 키워드를 추출합니다. 우선순위:

1. 구체적 섹터: 반도체, 빅테크, AI, 로봇, 우주항공, 방산, 배당
2. 자산유형: 국채, 채권, 금리
3. 지역: 인도, 베트남, 중국, 미국, 글로벌, 국내

매핑 테이블 (`backend/app/services/news_service.py`의 `KEYWORD_MAP`):

| 상품명 키워드 | 검색어 |
|-------------|--------|
| 반도체 | 반도체주 |
| 빅테크 | 미국 빅테크 |
| AI | AI주식 |
| 로봇 | 로봇주 |
| 우주항공 | 우주항공 방산 |
| 국채 | 미국국채 금리 |
| 채권 | 채권시장 |
| 인도 | 인도증시 |
| 베트남 | 베트남증시 |
| 미국 | 미국증시 |
| 국내 | 코스피 |

## DB 테이블

`market_analysis_cache`
- `keyword` (unique, indexed): 검색 키워드
- `news_summary`: 뉴스 원문 요약 (최대 2000자)
- `analysis`: Gemini 시황 분석 결과
- `created_at`: 생성 시각 (48시간 후 만료)

## API

- Naver News API: 키워드별 뉴스 5건 검색 (Client ID/Secret 필요)
- Gemini API: 시황 분석 생성 + 코멘트 조합

## 코멘트 작성 원칙

- 객관적이면서 논리적으로 작성
- 고객이 충분히 납득할 수 있도록 구체적 설명
- 전문 용어는 쉽게 풀어서 설명
- 단정적 표현 대신 신중한 표현 사용 ('~할 수 있습니다', '~로 판단됩니다')
- 뉴스 출처를 직접 인용하지 않고 자연스럽게 녹여서 작성

## 관련 파일

- `backend/app/services/news_service.py` — 네이버 뉴스 검색 + 키워드 추출
- `backend/app/services/market_analysis_service.py` — 시황 분석 생성/캐시/프롬프트 조합
- `backend/app/models/market_analysis_cache.py` — DB 캐시 모델
- `backend/app/api/v1/reports.py` — AI 코멘트 API 엔드포인트
- `backend/.env` — NAVER_CLIENT_ID, NAVER_CLIENT_SECRET, GEMINI_API_KEY
